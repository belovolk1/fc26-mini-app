# Что нужно для начала создания проекта

Чек-лист: аккаунты, сервисы и порядок действий.

---

## 1. Telegram

### 1.1 Создать бота
- Откройте [@BotFather](https://t.me/BotFather) в Telegram.
- Отправьте команду: `/newbot`.
- Введите имя бота (например: `FC26 Ladder`) и username (например: `fc26_ladder_bot`).
- **Сохраните токен** вида `123456789:ABCdefGHI...` — он понадобится для бэкенда и Mini App.

### 1.2 Настроить Mini App (позже)
- В BotFather: `/mybots` → выберите бота → **Bot Settings** → **Menu Button** или **Configure**.
- Поле **Web App URL** заполните после того, как задеплоите фронт (например, ссылка от Vercel).

---

## 2. GitHub

### 2.1 Аккаунт и репозиторий
- Если нет аккаунта: [github.com](https://github.com) → Sign up.
- Создайте репозиторий для проекта (например `fc26-mini-app`).
- Локально в папке проекта выполните:
  ```bash
  git init
  git remote add origin https://github.com/ВАШ_ЛОГИН/fc26-mini-app.git
  ```

### 2.2 Зачем
- Хранение кода.
- Подключение к бесплатному хостингу (Vercel/Netlify) для авто-деплоя при каждом `git push`.

---

## 3. Хостинг фронта (Mini App) — бесплатно

Выберите **один** сервис и привяжите к нему GitHub-репозиторий.

| Сервис | Регистрация | Что делать |
|--------|-------------|------------|
| **Vercel** | [vercel.com](https://vercel.com) → Sign up (через GitHub) | New Project → Import из GitHub → выбрать репо → Deploy. После каждого push — новый деплой. |
| **Netlify** | [netlify.com](https://netlify.com) → Sign up (через GitHub) | Add new site → Import from Git → GitHub → выбрать репо → Deploy. |
| **Cloudflare Pages** | [dash.cloudflare.com](https://dash.cloudflare.com) → Pages → Create project | Connect to Git → GitHub → выбрать репо. |

**Важно:** после первого деплоя скопируйте URL сайта (например `https://fc26-mini-app.vercel.app`) — его нужно будет указать в BotFather как Web App URL.

## 4. Бэкенд и база данных — бесплатные варианты

Для ладдера, турниров, ELO и матчмейкинга нужны API и БД.

### 4.1 База данных (выберите один вариант)
| Сервис | Регистрация | Лимиты бесплатно |
|--------|-------------|------------------|
| **Supabase** | [supabase.com](https://supabase.com) → Sign up (GitHub) | PostgreSQL, авторизация, реальное время (WebSocket). Удобно для старта. |
| **PlanetScale** | [planetscale.com](https://planetscale.com) | MySQL, бесплатный план. |
| **Neon** | [neon.tech](https://neon.tech) | PostgreSQL, бесплатный tier. |
| **Railway** | [railway.app](https://railway.app) | Можно поднять и БД, и бэкенд; есть бесплатный лимит. |

### 4.2 Бэкенд (API + матчмейкинг + бот)
| Вариант | Где хостить | Регистрация |
|---------|-------------|-------------|
| **Node.js (Express/Fastify)** | Railway, Render, Fly.io | У каждого свой sign up; привязка к GitHub для авто-деплоя. |
| **Supabase + Edge Functions** | Supabase | Уже в пункте 4.1; логику можно частично перенести в Edge Functions. |
| **Отдельный сервер бота** | Railway / Render / свой VPS | Тот же репо или отдельный; бот слушает обновления и держит WebSocket для матчмейкинга. |

**Рекомендация для старта:** Supabase (БД + авторизация по Telegram + при необходимости Edge Functions) + отдельный маленький бэкенд на Railway/Render для бота и очереди матчмейкинга (WebSocket).

## 6. Порядок действий (кратко)

1. **Telegram:** создать бота в BotFather, сохранить токен.
2. **GitHub:** создать репозиторий, сделать `git init` и `git remote add origin ...` в папке проекта.
3. **Локально:** установить Node.js и Git.
4. **Хостинг фронта:** зарегистрироваться на Vercel (или Netlify), подключить репо, сделать первый деплой (когда появится код фронта).
5. **БД:** зарегистрироваться в Supabase (или другом сервисе), создать проект, сохранить URL и ключ (anon key).
6. **Бэкенд:** позже — создать проект на Railway/Render, привязать репо с бэкендом, прописать переменные (токен бота, ссылка на БД).
7. **BotFather:** после деплоя фронта указать Web App URL в настройках бота.

---

## 7. Что хранить в секрете (не коммитить в Git)

- Токен Telegram-бота.
- URL и ключи к БД (Supabase и т.п.).
- Секретные ключи API.

Используйте **переменные окружения** (`.env` локально, в Vercel/Railway — настройки Environment Variables). В репозиторий добавьте файл `.env.example` с подсказками вида `TELEGRAM_BOT_TOKEN=your_token_here`, без реальных значений.
